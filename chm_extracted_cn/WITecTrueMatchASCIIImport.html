WITec ASCII导入

您可以使用以下WITec ASCII格式导入一个或多个光谱，以执行搜索或将光谱添加到自定义数据库。


文件必须以[WITEC_TRUEMATCH_ASCII_HEADER]开头。


每个光谱必须以[SpectrumHeader]开头，包含关于光谱的基本信息。


可选地，可以定义[SampleMetaData]部分，以便在将光谱添加到自定义数据库时添加一些样品属性。格式为<key> = <value>，如果前面有"double"或"int"，您可以定义一个浮点数或整数作为值。


最后，必须跟随[SpectrumData]数据部分，每行一个光谱值，从xdata/光谱位置（单位为nm或1/cm）开始，后跟<制表符>字符，再后跟该像素的ydata/CCD计数。浮点数格式必须使用"."作为小数分隔符。


示例：


[WITEC_TRUEMATCH_ASCII_HEADER]


[SpectrumHeader]

Title = Component 1

ExcitationWavelength = 532.235

SpectrumSize = 1024

XDataKind = nm (or 1/cm)


[SampleMetaData]

CAS_NUMBER = 123-54578

double mol_weight = 63.123

int NumComponents = 2


[SpectrumData]

YourXData1 <tabulator> YourYData1

YourXData2 <tabulator> YourYData2

....

YourXData1024 <tabulator> YourYData1024


[SpectrumHeader]

Title = Component 2

ExcitationWavelength = 785.122

SpectrumSize = 1600

XDataKind = 1/cm


[SampleMetaData]

CAS_NUMBER = 123-11222

double mol_weight = 11.55


[SpectrumData]

YourXData1 <tabulator> YourYData1

YourXData2 <tabulator> YourYData2

....

YourXData1600 <tabulator> YourYData1600


...

...

...




共享XData


对于共享相同X轴值/XData的大量光谱，也可以只在第一个[SpectrumHeader]之前定义一次XData。这样，可以缩短ASCII保存/加载时间。在这种情况下，每个光谱只需要定义YData。XData值的数量必须为SpectrumSize，并且所有光谱必须具有相同的大小。

示例：


[WITEC_TRUEMATCH_ASCII_HEADER]


[XData]

YourXData1

YourXData2

...

YourXData1600


[SpectrumHeader]

...

SpectrumSize = 1600

...
