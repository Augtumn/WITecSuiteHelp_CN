MATLAB

可以使用 MATLAB 与 WITec Control 的 COM 自动化配合，但存在几个问题。

  1. MATLAB 需要 ProgID 才能连接到 COM 服务器。默认未设置。在注册表中添加以下条目：

[HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Witec.COMAutomation]

[HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Witec.COMAutomation\CLSID]

@="{C45E77CE-3D66-489A-B5E2-159F443BD1AA}"

[HKEY_LOCAL_MACHINE\SOFTWARE\Classes\WOW6432Node\CLSID\{C45E77CE-3D66-489A-B5E2-159F443BD1AA}\ProgID]

@="Witec.COMAutomation"


  1. COM 接口返回 IUnknown 接口，需要将其转换为适当的接口。这仅在 32 位版本的 MATLAB 中可行。最后一个可用的 32 位版本是 R2015b。在 64 位 MATLAB 中，示例代码的最后一行会抛出错误，因此无法使用子系统。（请参考actxserver的帮助：64 位 MATLAB 不支持自定义接口。）
  2. 连接时无法定义服务器类型。默认情况下，MATLAB 将作为进程内服务器连接，但需要作为进程外服务器连接。
    • 对于 32 位 MATLAB：删除以下注册表项，并在 MATLAB 连接后恢复它。

HKEY_CLASSES_ROOT\WOW6432Node\CLSID\{C45E77CE-3D66-489A-B5E2-159F443BD1AA}\InprocServer32


    • 对于 64 位 MATLAB：这不是问题，因为它无法将 32 位 DLL 加载为进程内服务器，而是将其作为进程外服务器使用。


由于这些原因，不建议直接使用 MATLAB。一种解决方案是创建 C 包装器来实现对 MATLAB 的函数调用。


示例代码：

witeccom = actxserver('Witec.COMAutomation')

witeccom.RequestWriteAccess(true)

witeccom.HasWriteAccess

witeccore = invoke(witeccom, 'IBUCSCore')

subsyslist = witeccore.GetSubSystemsList('',0)

inttime = witeccore.GetSubSystemDefaultInterface('UserParameters|SequencerAsTimeGoesBy|IntegrationTime')

inttimefloat = inttime.invoke('IBUCSFloat')
