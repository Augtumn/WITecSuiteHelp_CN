COM 接口

本节列出所有可用的接口及其各自的方法。实现这些接口的类在类部分中描述。

所有接口的命名以IBUCS开头（Interface Basic Universal COM Server）。所有接口都从 IDispatch 派生。


IBUCSCore

读取其层次结构中可用的子系统对象。

GetSubSystemsList(in Name : BSTR, in SubSystemDepth : int) : IBUCSSubSystemList

描述        

列出从指定子系统开始的所有子系统，并以列表形式返回。

参数

Name：起始子系统的路径。使用空字符串从根开始。

SubSystemsDepth：定义应列出的层级数。如果为 0，则仅列出一个层级（已定义子系统的子系统）。

返回值

创建一个 CBUCSSubSystemsList 对象并返回 IBUCSSubSystemsList 指针。

GetSubSystem(in Name : BSTR, in refid : IID) : IUnknown

描述        

创建一个子系统对象并使用指定的接口。

参数

Name：子系统的路径。

refid：应用于子系统的接口的 GUID。该接口应为请求的子系统实现。

返回值

如果子系统的路径有效且接口已实现，则返回定义的接口作为 IUnknown*。需要将其转换为根据指定 IId 对应的接口。

GetSubSystemEx(in Name : BSTR, in InterfaceName : BSTR) : IUnknown

描述        

创建一个子系统对象，但接受接口 GUID 作为字符串。

参数

Name：子系统的路径。

InterfaceName：应应用于子系统的接口 GUID 作为字符串（格式如末尾表格所示）。该接口应为请求的子系统实现。

返回值

如果子系统的路径有效且接口已实现，则返回定义的接口作为 IUnknown*。需要将其转换为根据指定 IId 对应的接口。

GetSubSystemDefaultInterface(in Name : BSTR) : IUnknown

描述        

创建一个子系统对象并使用默认接口。

参数

Name：子系统的路径。

返回值

如果子系统的路径有效，则返回默认接口（根据 IDL 定义）作为 IUnknown*。需要将其转换为默认接口。


IBUCSSubSystemsList

CBUCSSubSystemsList的接口，其中包含子系统列表。

GetNumberOfSystems() : int

描述        

给出列表中的元素数量。

参数

-

返回值

列表中子系统的数量。

GetBaseName() : BSTR

描述        

检索所有子系统的基名称。

参数

-

返回值

列表中所有子系统的路径。（该路径用于创建列表。）

GetSubSystemName(in SystemIndex : int) : BSTR

描述        

返回由 SystemIndex 指定的子系统的名称。

参数

SystemIndex：子系统的索引。必须 >= 0 且 < 子系统数量（GetNumberOfSystems() - 1）

返回值

包含子系统名称的字符串。

GetSystemSubSystemsList(in SystemIndex : int) : IBUCSSubSystemList

描述        

创建 SystemIndex 指定子系统下的子系统列表。

参数

SystemIndex：子系统的索引。必须 >= 0 且 < 子系统数量（GetNumberOfSystems() - 1）

返回值

创建一个 CBUCSSubSystemsList 对象，其中包含指定子系统下的子系统，并返回其 IBUCSSubSystemsList 指针。

GetSubSystemNameAndIId(in SystemIndex : int, out SystemName : BSTR, out DefaultInterfaceId : BSTR) : HRESULT

描述        

返回由 SystemIndex 指定的子系统的名称和 GUID 字符串。

参数

SystemIndex：子系统的索引。必须 >= 0 且 < 子系统数量（GetNumberOfSystems() - 1）

SystemName：一个字符串指针，填入子系统的名称

DefaultInterfaceId：一个字符串指针，填入子系统默认接口的 GUID

返回值

成功则返回 true。


IBUCSAccess

此接口控制客户端对服务器的访问。

HasReadAccess() : bool

描述        

确定客户端是否具有读权限。

参数

-

返回值

如果对象在服务器上有读权限，则返回 true。

HasWriteAccess() : bool

描述        

确定客户端是否具有写权限。

参数

-

返回值

如果对象在服务器上有写权限，则返回 true。如果应用程序的一个对象在服务器上有写权限，则该应用程序创建的所有对象都有写权限。

RequestWriteAccess(in WantWriteAccess : bool) : HRESULT

描述        

请求客户端的写权限。

参数

WantWriteAccess：true 表示客户端想获取写权限，false 表示客户端想放弃写权限。服务器确保只有一个应用程序具有写权限。

返回值

如果在调用后对象在服务器上有写权限，则返回 true。

IBUCSEnum

此接口可用于更改或读取列表或枚举参数。列表参数对每个数值都有相应的字符串。

GetAvailableValues(out NumericalValues : SAFEARRAY, out StringValues : SAFEARRAY) : int

参数

NumericalValues：数组指针，填入此参数的数值。

String Values：数组指针，填入相应数值的字符串。

返回值

返回可用值的数量。

C/C++/C#：使用此函数后需要为 NumericalValues 和 StringValues 调用SafeArrayDestroy()！

GetValue(out ValueString : BSTR) : int

参数

ValueString：一个字符串指针，由服务器填入当前选中值的相应字符串。

返回值

参数的当前数值。

SetValueNumeric(in Value : int) : HRESULT

参数

Value：应由参数设置的数值。主机应用程序需要确保该值在可能的范围内。服务器 DLL 不会检查。

返回值

成功则返回 true。

SetValueString(in Value : BSTR) : HRESULT

参数

Value：应由参数设置的值的相应字符串。主机应用程序需要确保该值有效。服务器 DLL 不会检查。

返回值

成功则返回 true。

IBUCSStatusContainer

一个用于从服务器检索状态信息的接口。这可能是不同类型的单个值或一个或多个维度的数组。

Update() : HRESULT

更新容器的状态。

参数

-

返回值

成功则返回 true。

GetSingleValueAsString(out StringValue : BSTR) : bool

参数

StringValue：指向字符串的指针，该字符串由容器当前状态的字符串填充。状态不允许是数组。

返回值

成功则返回 true。

GetSingleValueAsInt(out IntValue : int) : bool

参数

IntValue：指向 Int 值的指针，该值由容器当前状态的值填充。状态不允许是数组。

返回值

成功则返回 true。

GetSingleValueAsDouble(out DoubleValue : double) : bool

参数

DoubleValue：指向 double 值的指针，该值由容器当前状态的值填充。状态不允许是数组。

返回值

成功则返回 true。

GetStatusArray(out Dimensions : int, out DimensionExtents : SAFEARRAY, out StatusArray : SAFEARRAY) : bool

参数

Dimensions：指向 Int 值的指针，调用后包含状态数组的维数。

DimensionsExtents：指向数组的指针，调用后包含每个维度范围的一维数组。

Status Array：指向数组的指针，调用后包含完整的状态数组。注意保持一致的索引。

返回值

如果状态数组有效，则返回 true。

C/C++/C#：使用此函数后需要为 DimensionExtents 和 StatusArray 调用SafeArrayDestroy()！

GetStatusProperties(out Caption : BSTR, out Unit : BSTR) : bool

参数

Caption：指向字符串的指针，填入当前状态值的描述。

Unit：指向字符串的指针，填入当前状态值的单位。

返回值

如果状态有效，则返回 true。


IBUCSFillStatusContainer

一个用于将一维数据数组传输到服务器的接口。

FillDataArray(in DataArray : SAFEARRAY) : HRESULT

参数

DataArray：包含数据的数组（variant SAFEARRAY，数据类型为 VT_ARRAY | VT_R4 或 VT_ARRAY | VT_BSTR）

返回值

如果数组传输成功，则返回 true。

C/C++/C#：服务器会为传递的 DataArray 调用VariantClear()。
